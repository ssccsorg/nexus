FIH Blackboard Core Restructuring

Published

May 15, 2026

Summary

Comprehensive restructuring of the FIH Blackboard implementation: extracted core interfaces, unified persistence behind a storage trait, and validated through multi-agent stress testing.

Structural Changes

Layout

before:                          after:
workers/                         gateway/         (agent entry point)
engines/                         ext/             (external backends)
core/graph (monolithic)          core/api/        (Blackboard trait + FIH)
core/cypher (separate)           core/graph/      (GraphBlackboard + cypher)
core/index (skeleton)            core/table/        (SqliteStorage)
  • workers/ -> gateway/: agent entry and routing layer
  • engines/ -> ext/: external FIH backends, flattened with shared _runner/
  • core/api/: pure Blackboard trait + Fact/Intent/Hint/FihHash, no storage deps
  • core/graph/: GraphBlackboard (petgraph) + GraphAccess + Cypher executor (merged from core/cypher/)
  • core/table/: SqliteStorage implementing Storage trait
  • core/index/ removed: vector search is graph-level operation

Key Interfaces

  • Blackboard trait: 8 methods (submit_fact, submit_intent, claim_intent, heartbeat, release_intent, conclude_intent, read_state, submit_hint)
  • Storage trait: 2 methods (log_fih, load_events), thread-safe (Send + Sync)
  • GraphAccess trait: 10 methods, petgraph-specific, used by Cypher executor
  • GraphBlackboard::with_storage(): inject any Storage impl at runtime

  • Bug Fix Pipeline: reporter -> triager -> dev -> reviewer

  • SSCCS Primitive Discovery: 5 agents formalize Segment from 3 IR observations

Key Discoveries

  1. Append-only log = complete determinism: Any prior state exactly reconstructed by replay. No snapshots needed.

  2. Storage trait as system boundary: Same FIH logic across SQLite, blockchain, or memory. Inside = deterministic; outside = implementation.

  3. read_state() = observation: Field evolution is observation accumulation. No separate mechanism.

  4. Parallelism is structural: 10,000 parallel events validated – 50 threads, zero conflicts.

  5. Hints = Field constraints: Governance, focus, safety all carried by Hint primitive.
